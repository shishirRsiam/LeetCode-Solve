# গসাগু (GCD) ও লসাগু (LCM) – সম্পূর্ণ নির্দেশিকা ও ডকুমেন্টেশন

**শ্রেণি:** প্রোগ্রামিং ও গণিত (নবম-দ্বাদশ ও বিশ্ববিদ্যালয়ের শিক্ষার্থীদের জন্য)  
**ভাষা:** বাংলা

## ১. ভূমিকা (Introduction)

গণিত এবং কম্পিউটার প্রোগ্রামিংয়ের জগতে **গসাগু** (গরিষ্ঠ সাধারণ গুণনীয়ক / GCD) এবং **লসাগু** (লঘিষ্ঠ সাধারণ গুণিতক / LCM) দুটি মৌলিক ও অত্যন্ত গুরুত্বপূর্ণ ধারণা। ভগ্নাংশের সরলীকরণ, সংখ্যাতত্ত্ব (Number Theory), ক্রিপ্টোগ্রাফি এবং বিভিন্ন অ্যালগরিদম ডিজাইনে এদের ব্যবহার রয়েছে। আজকের এই ডকুমেন্টেশনে আমরা এই দুটি ধারণার গাণিতিক সংজ্ঞা, তাদের পারস্পরিক সম্পর্ক এবং সবচেয়ে কার্যকরী ইউক্লিডীয় অ্যালগরিদম নিয়ে আলোচনা করব, পাশাপাশি **Python** ও **C++** ভাষায় তাদের বাস্তবায়ন দেখব।

## ২. গাণিতিক সংজ্ঞা (Mathematical Definition)

### ২.১. গসাগু (GCD - Greatest Common Divisor)
দুটি বা ততোধিক পূর্ণসংখ্যার **গরিষ্ঠ সাধারণ গুণনীয়ক** হল সেই বৃহত্তম ধনাত্মক পূর্ণসংখ্যা, যা দিয়ে প্রতিটি সংখ্যাকে নিঃশেষে (ভাগশেষ ০ রেখে) ভাগ করা যায়।

> **উদাহরণ:** ১২ ও ১৮-এর গুণনীয়কগুলো হলো:
> * ১২ = ১, ২, ৩, ৪, ৬, ১২
> * ১৮ = ১, ২, ৩, ৬, ৯, ১৮
> 
> সাধারণ গুণনীয়কগুলো হলো ১, ২, ৩, এবং ৬। এদের মধ্যে সবচেয়ে বড়টি হলো **৬**।  
> ∴ GCD(১২, ১৮) = ৬

### ২.২. লসাগু (LCM - Least Common Multiple)
দুটি বা ততোধিক পূর্ণসংখ্যার **লঘিষ্ঠ সাধারণ গুণিতক** হল সেই ক্ষুদ্রতম ধনাত্মক পূর্ণসংখ্যা, যা প্রতিটি সংখ্যা দ্বারা নিঃশেষে বিভাজ্য।

> **উদাহরণ:** ১২ ও ১৮-এর গুণিতকগুলো হলো:
> * ১২ এর গুণিতক: ১২, ২৪, ৩৬, ৪৮, ৭২...
> * ১৮ এর গুণিতক: ১৮, ৩৬, ৫৪, ৭২...
> 
> সাধারণ গুণিতকগুলো হলো ৩৬, ৭২... এদের মধ্যে সবচেয়ে ছোটটি হলো **৩৬**।  
> ∴ LCM(১২, ১৮) = ৩৬

## ৩. গসাগু ও লসাগুর মধ্যে সম্পর্ক (The Core Relation)

দুটি সংখ্যা \( a \) এবং \( b \)-এর জন্য একটি অত্যন্ত গুরুত্বপূর্ণ সূত্র রয়েছে, যা গণিতের মৌলিক সূত্র হিসেবে পরিচিত:

> \[
> \boxed{\text{GCD}(a, b) \times \text{LCM}(a, b) = a \times b}
> \]

অর্থাৎ, যদি আমরা গসাগু বের করতে পারি, তাহলে খুব সহজেই লসাগু বের করা যায়:

> \[
> \text{LCM}(a, b) = \frac{a \times b}{\text{GCD}(a, b)}
> \]

**সতর্কতা:** বাস্তব কোডে \( a \times b \) করার সময় যদি সংখ্যা দুটি খুব বড় হয়, তবে Integer Overflow (পূর্ণসংখ্যা সীমা অতিক্রম) হতে পারে। তাই কোডে সাধারণত **গসাগু দিয়ে ভাগ করে তারপর গুণ** করা হয় (অর্থাৎ, \( (a / \text{GCD}) \times b \))।

## ৪. গসাগু বের করার কার্যকরী পদ্ধতি: ইউক্লিডীয় অ্যালগরিদম (Euclidean Algorithm)

গসাগু বের করার সবচেয়ে পুরনো ও দ্রুততম পদ্ধতি হলো **ইউক্লিডীয় অ্যালগরিদম**। এটি এই সরল নীতিতে কাজ করে:

> \[
> \text{GCD}(a, b) = \text{GCD}(b, a \mod b)
> \]
> যতক্ষণ না \( b = 0 \) হয়। যখন \( b = 0 \), তখন \( a \)-ই হবে গসাগু।

**পদক্ষেপসমূহ (Steps):**
1.  যদি `b` এর মান ০ হয়, তাহলে `a` রিটার্ন কর।
2.  অন্যথায়, `a`-কে `b` দ্বারা ভাগ করে ভাগশেষ (`a % b`) বের কর।
3.  এখন নতুন করে `a = b` এবং `b = ভাগশেষ` সেট কর।
4.  ধাপ ১-এ ফিরে যাও।

**উদাহরণ (১২ ও ১৮):**
*   GCD(১২, ১৮) → GCD(১৮, ১২%১৮ = ১২) → GCD(১৮, ১২)
*   GCD(১৮, ১২) → GCD(১২, ১৮%১২ = ৬) → GCD(১২, ৬)
*   GCD(১২, ৬) → GCD(৬, ১২%৬ = ০) → GCD(৬, ০)
*   যেহেতু b=০, তাই উত্তর = **৬**।

**Time Complexity (সময় জটিলতা):** \( O(\log \min(a, b)) \) – যা অত্যন্ত দ্রুত।

## ৫. Python এ কোড বাস্তবায়ন (Implementation in Python)

পাইথনে ইউক্লিডীয় অ্যালগরিদম লেখা অত্যন্ত সহজ। পাইথনের টাপল (Tuple) আনপ্যাকিং সুবিধা ব্যবহার করে আমরা খুব কমপ্যাক্ট কোড লিখতে পারি।

```python
# ১. গসাগু (GCD) বের করার ফাংশন - ইউক্লিডীয় অ্যালগরিদম (ইটারেটিভ)
def gcd(a, b):
    while b != 0:
        a, b = b, a % b  # একই সাথে a তে b এবং b তে ভাগশেষ বসিয়ে দিচ্ছি
    return abs(a)  # নেগেটিভ সংখ্যার জন্য Absolute value রিটার্ন করি

# ২. লসাগু (LCM) বের করার ফাংশন
def lcm(a, b):
    if a == 0 or b == 0:
        return 0  # শূন্য দিয়ে ভাগ না করার জন্য চেক
    return abs(a // gcd(a, b) * b)  # Overflow এড়াতে আগে ভাগ, পরে গুণ

# ---------- মূল প্রোগ্রাম (Driver Code) ----------
if __name__ == "__main__":
    num1 = 12
    num2 = 18
    
    g = gcd(num1, num2)
    l = lcm(num1, num2)
    
    print(f"সংখ্যা দুটি: {num1} ও {num2}")
    print(f"গসাগু (GCD): {g}")
    print(f"লসাগু (LCM): {l}")
    
    # সম্পর্ক যাচাই
    print(f"\nযাচাই: GCD * LCM = {g} * {l} = {g * l}")
    print(f"a * b = {num1} * {num2} = {num1 * num2}")
```

**আউটপুট:**
```
সংখ্যা দুটি: 12 ও 18
গসাগু (GCD): 6
লসাগু (LCM): 36

যাচাই: GCD * LCM = 6 * 36 = 216
a * b = 12 * 18 = 216
```

## ৬. C++ এ কোড বাস্তবায়ন (Implementation in C++)

C++-এ কোড করার সময় আমাদের অবশ্যই **ইন্টিজার ওভারফ্লো** সম্পর্কে সচেতন থাকতে হবে। তাই লসাগু বের করার সময় \( (a / gcd) \times b \) এই ফর্মুলা ব্যবহার করা নিরাপদ। এছাড়াও C++17 স্ট্যান্ডার্ডে `<numeric>` হেডারে `std::gcd` আছে, তবে আমরা নিজেরা ফাংশন বানিয়ে দেখাবো।

```cpp
#include <iostream>
#include <cstdlib> // abs() ফাংশনের জন্য
using namespace std;

// ১. গসাগু (GCD) বের করার ফাংশন - ইউক্লিডীয় অ্যালগরিদম (ইটারেটিভ)
int gcd(int a, int b) {
    a = abs(a); // নেগেটিভ সংখ্যা হ্যান্ডেল করার জন্য
    b = abs(b);
    
    while (b != 0) {
        int temp = b;
        b = a % b;
        a = temp;
    }
    return a;
}

// ২. লসাগু (LCM) বের করার ফাংশন
int lcm(int a, int b) {
    if (a == 0 || b == 0) {
        return 0; // শূন্য দিয়ে ভাগ না করার জন্য চেক
    }
    a = abs(a);
    b = abs(b);
    
    // ওভারফ্লো এড়ানোর জন্য (a / gcd) আগে ভাগ, তারপর b দিয়ে গুণ
    return (a / gcd(a, b)) * b;
}

// ---------- মূল প্রোগ্রাম (Driver Code) ----------
int main() {
    int num1 = 12;
    int num2 = 18;
    
    int g = gcd(num1, num2);
    int l = lcm(num1, num2);
    
    cout << "সংখ্যা দুটি: " << num1 << " ও " << num2 << endl;
    cout << "গসাগু (GCD): " << g << endl;
    cout << "লসাগু (LCM): " << l << endl;
    
    // সম্পর্ক যাচাই
    cout << "\nযাচাই: GCD * LCM = " << g << " * " << l << " = " << g * l << endl;
    cout << "a * b = " << num1 << " * " << num2 << " = " << num1 * num2 << endl;
    
    return 0;
}
```

**আউটপুট:**
```
সংখ্যা দুটি: 12 ও 18
গসাগু (GCD): 6
লসাগু (LCM): 36

যাচাই: GCD * LCM = 6 * 36 = 216
a * b = 12 * 18 = 216
```

## ৭. একাধিক সংখ্যার জন্য গসাগু ও লসাগু (Extension for Multiple Numbers)

যদি দুইয়ের বেশি সংখ্যার গসাগু বের করতে হয়, তাহলে **জোড়ায় জোড়ায়** (Pairwise) বের করতে হবে:

> \[
> \text{GCD}(a, b, c) = \text{GCD}(\text{GCD}(a, b), c)
> \]

একইভাবে লসাগুর ক্ষেত্রেও:
> \[
> \text{LCM}(a, b, c) = \text{LCM}(\text{LCM}(a, b), c)
> \]

**উদাহরণ (Python):**
```python
from functools import reduce

numbers = [12, 18, 24]
gcd_all = reduce(gcd, numbers)  # 12,18 = 6 ; 6,24 = 6
lcm_all = reduce(lcm, numbers)  # 12,18 = 36 ; 36,24 = 72
```

## ৮. উপসংহার ও করণীয় (Conclusion & Best Practices)

1.  **গসাগু বের করতে সবসময় ইউক্লিডীয় অ্যালগরিদম ব্যবহার করবেন**। এটি পুনরাবৃত্তিমূলক (Iterative) বা পুনর্বার (Recursive) দুইভাবেই লেখা যায়। পুনরাবৃত্তিমূলক পদ্ধতি স্ট্যাক ওভারফ্লো ঝুঁকি কমায়, তাই তা বেশি নিরাপদ।
2.  **ওভারফ্লো এড়িয়ে চলুন**: লসাগু বের করার সময় \( (a \times b) / \text{GCD} \) না লিখে \( (a / \text{GCD}) \times b \) লিখুন।
3.  **নেগেটিভ সংখ্যা**: বেশিরভাগ অ্যালগরিদম ধনাত্মক সংখ্যার জন্য কাজ করে। ইনপুট নেওয়ার সময় `abs()` ফাংশন ব্যবহার করে সংখ্যাকে ধনাত্মকে রূপান্তর করে নিন।
4.  **বিল্ট-ইন ফাংশন**: C++17-এ `std::gcd` এবং `std::lcm` ( `<numeric>` ) আছে এবং Python 3.9-এ `math.gcd` আছে। তবে ভিত্তি বুঝে নিজে কোড করতে পারাটাই সবচেয়ে গুরুত্বপূর্ণ।

আশা করি এই ডকুমেন্টেশনটি তোমাদের গসাগু ও লসাগু সম্পর্কে সম্পূর্ণ ধারণা দিতে পেরেছে। কোডগুলো নিজে চালিয়ে দেখ এবং বিভিন্ন ইনপুট দিয়ে যাচাই কর।
